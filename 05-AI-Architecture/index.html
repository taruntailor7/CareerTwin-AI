AI Architecture Specification

Decision Intelligence AI System

CareerTwin AI uses a modular AI architecture: multi-source profile understanding, structured reasoning, evidence-backed simulation, and strict safety controls. Every recommendation is traceable, calibrated, and continuously evaluated.

End-to-End AI Architecture Diagram

Career Twin Engine

  • Builds multi-dimensional user twin: skills, motivations, constraints, risk profile, growth vector.
  • Versioned snapshots with immutable IDs for reproducible simulations.
  • Supports recalibration triggers from new data or behavior changes.

Resume Parser

  • Input formats: PDF/DOCX/text.
  • Pipeline: OCR/parse → section segmentation → entity extraction → role timeline normalization.
  • Outputs structured JSON with per-field confidence and source offsets.

LinkedIn Analysis

  • Extracts roles, tenure, skill endorsements, growth signals, and network context.
  • Reconciles with resume and user-edited profile; conflict resolver requests explicit user confirmation.
  • Produces quality score indicating completeness and freshness.

GitHub Analysis

  • Extracts language mix, contribution cadence, collaboration density, repository complexity proxies.
  • Private repository inclusion is opt-in; aggregation supports privacy-first mode.
  • Outputs technical signature vector for twin enrichment.

Portfolio Analysis

  • Crawls portfolio/project pages for impact statements, scope, outcomes, and role descriptions.
  • Classifies evidence into depth, leadership, creativity, and business impact.
  • Supports user review and exclusion of noisy extracted entries.

Career Interview Engine

  • Adaptive question graph driven by confidence gaps and contradiction detection.
  • Maintains conversational memory per session with summarization checkpoints.
  • Labels responses: confirmed, inferred, uncertain.

Memory

Short-term Memory

  • Session context for interview and simulation runtime decisions.
  • Windowed summary to keep prompt size bounded.

Long-term Memory

  • Versioned twin snapshots and decision history.
  • Retrieval by recency, relevance, and source reliability.

Context Builder

  • Aggregates only relevant slices of memory + profile + simulation assumptions.
  • Applies token budgeting and ordered context packing by priority.
  • Injects system guardrails and output schema contracts before model call.

Simulation Engine

  • Generates multi-path projections using user constraints and market signals.
  • Computes: salary growth, skill growth, promotion probability, burnout risk, optionality, work-life balance.
  • Runs sensitivity checks across assumption perturbations to avoid brittle outputs.

Evidence Engine

  • Attaches provenance metadata to every recommendation statement.
  • Source classes: user-stated, imported, inferred, external market source.
  • Blocks claims without sufficient support and downgrades confidence where evidence is weak.

Confidence Engine

  • Combines data quality, source agreement, model stability, and scenario sensitivity.
  • Outputs confidence bands per metric and per final recommendation.
  • Supports calibration tracking against future outcome feedback.

Reasoning Pipeline

  1. Intent & objective extraction
  2. Context assembly and normalization
  3. Scenario generation and feasibility check
  4. Metric evaluation and tradeoff synthesis
  5. Evidence linking and confidence scoring
  6. Safety and policy filters
  7. Structured response rendering

Prompt Flow

Function Calling

  • Tool contracts enforce typed schemas for parsing, retrieval, simulation, evidence lookup, and report generation.
  • All tool responses validated before feeding back into reasoning pipeline.
  • Idempotency keys required for mutating operations.

Safety

  • Policy filter for harmful advice patterns and high-risk deterministic statements.
  • Output guardrails ensuring uncertainty language for all predictive content.
  • Escalation pathways for sensitive scenarios and user support cues.

Hallucination Prevention

  • Evidence-required generation mode: unsupported statements are blocked or marked uncertain.
  • Consistency checks across sources before finalizing output.
  • Self-critique stage: model validates claim-evidence alignment before response release.

Prompt Injection Protection

  • Untrusted external content (portfolio/web/import text) sandboxed and stripped of instruction-like patterns.
  • Hard separation between system instructions and retrieved user content.
  • Policy scanner to detect jailbreak prompts, hidden instructions, and malicious payload tokens.

Evaluation Framework

Offline Evals

  • Golden datasets for profile parsing accuracy.
  • Simulation consistency tests across repeated runs.
  • Reasoning quality scoring with rubric-based review.

Online Evals

  • A/B tests on clarity, trust, and decision activation metrics.
  • Calibration tracking between confidence and real outcomes.
  • User-reported usefulness and correction feedback loops.

Monitoring

  • Operational: latency, error rates, token usage, tool call failures.
  • Quality: evidence coverage rate, contradiction detection rate, confidence calibration drift.
  • Safety: policy violation counts, blocked outputs, injection detection events.
  • Product impact: simulation completion rate, decision activation, report generation success.

Release gates require passing minimum thresholds in safety, consistency, and calibration before production rollout.
