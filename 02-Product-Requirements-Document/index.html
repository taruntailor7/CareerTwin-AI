Product Requirements Document

CareerTwin AI PRD

This document defines complete product requirements for CareerTwin AI, including feature behavior, constraints, business rules, and measurable acceptance standards.

Functional Requirements

FR-01 Twin Onboarding Orchestration

  • System shall provide guided onboarding with source imports and AI interview.
  • System shall maintain progress state and allow resume later.
  • System shall generate initial twin profile when minimum required signals are validated.

FR-02 Signal Ingestion

  • System shall ingest resume, LinkedIn, GitHub, and portfolio data independently.
  • System shall allow users to skip non-essential sources and continue.
  • System shall label each data point by source provenance.

FR-03 Twin Model Synthesis

  • System shall map inputs into skill, motivation, preference, and risk dimensions.
  • System shall expose confidence level per dimension.
  • System shall store twin versions as immutable snapshots.

FR-04 Simulation Engine

  • System shall run multiple scenario simulations from user-defined assumptions.
  • System shall compute tradeoffs (growth, earnings, burnout, optionality).
  • System shall return comparative outputs and recommended experiments.

FR-05 Insights and Reporting

  • System shall deliver weekly insights and trigger-based alerts.
  • System shall generate shareable reports with privacy controls.
  • System shall track decision history and outcome review entries.

FR-06 Configuration and Control

  • System shall provide user controls for data sources and deletions.
  • System shall allow AI response style preferences.
  • System shall support explicit consent for each integration.

Non-functional Requirements

ID Requirement Target

NFR-02 Availability 99.5% minimum for authenticated user flows
NFR-03 Security and privacy Explicit consent, encryption in transit and at rest, user deletion path
NFR-04 Explainability completeness 100% of AI recommendations include assumptions and evidence blocks
NFR-05 Accessibility WCAG 2.1 AA baseline for all core screens
NFR-06 Auditability All recommendation outputs retain version and source metadata

User Stories

US-01

As a career switcher, I want to compare three future paths so that I can choose one with clear tradeoffs.

US-02

As a time-constrained user, I want to import my resume and LinkedIn quickly so that I can avoid manual profile entry.

US-03

As a skeptical user, I want to inspect evidence behind AI outputs so that I can trust recommendations.

US-04

As an ambitious professional, I want weekly signals about timing and risk so that I can act before opportunities close.

US-05

As a user with changing goals, I want my twin to evolve over time so that recommendations stay relevant.

US-06

As a user discussing decisions with mentors, I want shareable reports so that I can get aligned feedback fast.

Business Rules

  • BR-01: AI must never output deterministic future claims.
  • BR-02: Recommendations require explicit assumptions and confidence bands.
  • BR-03: Imported data cannot overwrite user-confirmed data without user approval.
  • BR-04: Simulation runs require minimum profile completeness threshold.
  • BR-05: Users may disable any external source without losing account access.
  • BR-06: Share reports default to private and explicit opt-in for public link generation.

Acceptance Criteria

AC-01 Profile Completion

  • Given onboarding in progress, when mandatory fields are complete, then twin creation is enabled.
  • Given import failure, when retry succeeds, then progress continues without restart.

AC-02 Simulation Output

  • Given at least two scenarios, when simulation executes, then comparison output appears for all required dimensions.
  • Given missing assumptions, then system blocks final recommendation and requests additional inputs.

AC-03 Explainability

  • Given any recommendation card, then assumptions, evidence, tradeoffs, and confidence are visible.
  • Given source references, then each item indicates data origin type.

AC-04 Reporting

  • Given a selected simulation, when report export is triggered, then PDF-ready report is generated with privacy settings.
  • Given link sharing enabled, then link expiry setting is required.

Edge Cases

EC-01 Contradictory Inputs

User says "high stability" but selects founder scenario as primary. System must prompt tradeoff confirmation.

EC-02 Sparse Profile

User skips all imports and provides minimal interview data. System should run baseline simulation with low confidence warnings.

EC-03 Stale External Data

LinkedIn/GitHub data is outdated. System should display freshness indicators and ask for re-sync.

EC-04 High-risk Decision Bias

User repeatedly selects prestige paths despite mismatch. System should trigger reflective challenge prompts.

AI Behavior Requirements

  • AI-B01: Tone must be calm, precise, and non-judgmental.
  • AI-B02: AI must expose uncertainty and clarify weak data zones.
  • AI-B03: AI shall ask follow-up questions before high-impact recommendations if confidence is below threshold.
  • AI-B04: AI shall identify contradiction patterns across user goals and behavior.
  • AI-B05: AI shall output actionable experiments, not only abstract guidance.

Career Twin Lifecycle

Stage 1 - Genesis

Initial twin created via imports + interview with baseline confidence map.

Stage 2 - Calibration

Twin quality improves through corrections, simulations, and behavior signals.

Stage 3 - Decision Support

Twin powers major transition simulations and action planning.

Stage 4 - Evolution

Twin versions track trajectory shifts and outcome feedback loops.

Profile Creation Requirements

  • PC-01: Collect identity basics, goals, constraints, and preference inputs.
  • PC-02: Validate required fields with inline error recovery.
  • PC-03: Persist partial state autosave during onboarding.
  • PC-04: Show completion percentage and remaining steps.

Resume Import Requirements

  • RI-01: Support PDF/DOCX upload and parse to structured fields.
  • RI-02: Surface parse confidence per section.
  • RI-03: Allow user edit and confirmation before final save.
  • RI-04: Preserve original file for re-parse audit trail.

LinkedIn Import Requirements

  • LI-01: OAuth-based connection and explicit permission display.
  • LI-02: Import profile summary, roles, skills, and endorsements metadata.
  • LI-03: Highlight conflicts with existing data and request user choice.
  • LI-04: Show last sync timestamp and manual refresh action.

GitHub Import Requirements

  • GI-01: Connect GitHub account and ingest repository metadata.
  • GI-02: Derive language mix, contribution cadence, and collaboration signals.
  • GI-03: Support private repo exclusion by user control.
  • GI-04: Provide summary view before committing to twin profile.

Portfolio Import Requirements

  • PI-01: Accept portfolio URL and crawl project pages.
  • PI-02: Extract project outcomes, stack, and role indicators.
  • PI-03: Classify project evidence type: depth, leadership, creativity, impact.
  • PI-04: Let user discard low-quality extracted entries.

AI Interview Requirements

  • AII-01: Ask adaptive questions based on confidence gaps and contradictions.
  • AII-02: Support short-form and long-form response modes.
  • AII-03: Label responses as confirmed, inferred, or uncertain.
  • AII-04: End with summary confirmation before twin synthesis.

Career Simulations Requirements

  • CS-01: Build and compare at least 2 and up to 5 scenarios per run.
  • CS-02: Evaluate salary growth, skill growth, promotion probability, burnout risk, market demand, work-life balance, leadership growth.
  • CS-03: Display uncertainty bands and assumption impact.
  • CS-04: Generate path-specific next-step experiments.

Twin Evolution Requirements

  • TE-01: Maintain version history for all major profile or outcome shifts.
  • TE-02: Provide diff view between versions.
  • TE-03: Link simulation outcomes back to version state at execution time.
  • TE-04: Trigger recalibration prompts when profile drift is detected.

Career Insights Requirements

  • CI-01: Generate weekly insight brief with top opportunities and risks.
  • CI-02: Provide blind spot and capability gap alerts.
  • CI-03: Trigger timing alerts for market windows.
  • CI-04: Store insights in timeline for review and accountability.

Dashboard Requirements

  • DB-01: Display twin health, pending decisions, and top three next actions.
  • DB-02: Show latest simulation summary and confidence status.
  • DB-03: Surface overdue experiments and stale data warnings.
  • DB-04: Provide direct navigation to high-priority actions.

Reports Requirements

  • RP-01: Generate decision report with scenario comparisons and evidence.
  • RP-02: Support mentor view and executive summary view.
  • RP-03: Include configurable privacy redactions.
  • RP-04: Track report generation and share events for analytics.
