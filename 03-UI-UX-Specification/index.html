UI/UX Specification

Complete Screen and Experience Blueprint

This document defines the UI and UX behavior for every core CareerTwin AI screen: purpose, wireframe intent, components, interactions, states, accessibility, and flow continuity.

Global UX Patterns

Visual Language

  • Premium minimalist layout with strong hierarchy and calm whitespace.
  • Data-heavy areas use card clusters with progressive disclosure.
  • Core accent color reserved for primary actions and focus states.

Interaction Language

  • Primary action per screen; secondary actions visually subordinated.
  • Transition timing: 180ms for micro interactions, 280ms for screen transitions.
  • Never animate on error; keep failure states still and clear.

State Consistency

  • Skeleton loading for network-driven content blocks.
  • Empty states include explicit next action, never dead ends.
  • Success states include confirmation and suggested follow-up.

Accessibility Baseline

  • Keyboard support for all controls and dialog flows.
  • Minimum contrast ratio AA for text and actionable elements.
  • Reduced motion mode for non-essential animations.

Landing Page

Purpose

Convert high-intent visitors by demonstrating category differentiation in under 30 seconds.

Wireframe Description

Hero + interactive simulation teaser + trust band + social proof + CTA footer.

Components

Hero banner, scenario teaser widget, credibility strip, FAQ accordion, CTA panel.

Buttons

Primary: "Create Your Twin"; Secondary: "Watch 90-sec demo"; Tertiary: "How it works".

Inputs

Email field for waitlist/demo access and optional role selector for personalization.

Animations

Subtle constellation background parallax and hover lift on scenario cards.

Transitions

Hero-to-demo expansion animation with ease-out curve and scroll-linked opacity.

Loading States

Skeleton blocks for social proof cards and animated placeholder for teaser widget.

Empty States

If no social proof data, show curated narrative case examples.

Success States

Waitlist submission confirmation with estimated invite timeline.

Error States

Inline validation and retry prompt for failed lead capture submission.

Responsive Behaviour

Desktop split hero; mobile stacked flow with sticky bottom CTA.

Accessibility

Landmark structure, keyboard-navigable accordion, descriptive CTA labels.

User Flow

Discover → Understand value → Trust check → Start onboarding.

Authentication

Purpose

Enable secure account access with minimal onboarding friction.

Wireframe Description

Centered auth card with segmented tabs for sign in/sign up and SSO options.

Components

Auth form, social SSO buttons, legal consent text, progress reminder panel.

Buttons

Primary: Continue; Secondary: Continue with Google/LinkedIn; Link: Forgot password.

Inputs

Email, password, confirmation password, OTP field (if verification required).

Animations

Form state swap fade between sign-in and sign-up tabs.

Transitions

Post-auth route transition to onboarding with progress context chip.

Loading States

Button spinner with disabled submit to prevent duplicate requests.

Empty States

No external provider available state with fallback email auth.

Success States

Account verified message then auto-redirect to Twin Genesis.

Error States

Credential errors, expired OTP handling, locked account cooldown messaging.

Responsive Behaviour

Single-column card on mobile with large tap targets and full-width inputs.

Accessibility

Input labels, aria-invalid hints, error message association, focus retention.

User Flow

Entry point → Auth method selection → Validation → Onboarding start.

Create Career Twin

Purpose

Collect foundational data and initialize the first twin version.

Wireframe Description

Multi-step wizard with left progress rail and right context panel.

Components

Step navigator, data cards, progress meter, confidence status widget.

Buttons

Primary: Continue; Secondary: Save and exit; Utility: Skip optional step.

Inputs

Career goals, salary expectation range, location preference, work-style preferences.

Animations

Step completion check animation and twin graph node illumination.

Transitions

Horizontal step slide with history-preserving breadcrumb updates.

Loading States

Step-level loading while validating imported or parsed data.

Empty States

No data source connected fallback with manual input prompts.

Success States

Twin creation trigger confirmation and transition to reveal screen.

Error States

Step validation errors pinned near fields and summary at top.

Responsive Behaviour

Progress rail transforms into compact stepper on mobile.

Accessibility

Stepper announced with current step count and completion status.

User Flow

Profile basics → Imports → Preferences → Review → Twin generation.

AI Interview

Purpose

Fill ambiguity gaps and capture nuanced user motivations.

Wireframe Description

Conversational card interface with one prompt at a time and context memory panel.

Components

Prompt card, response composer, confidence indicator, summary tracker.

Buttons

Primary: Submit answer; Secondary: Clarify question; Utility: Skip for now.

Inputs

Free text, quick-select chips, optional voice transcript.

Animations

Prompt reveal fade; confidence meter pulse on answer submission.

Transitions

Question-to-question crossfade with conversation continuity marker.

Loading States

AI thinking indicator with deterministic timeout fallback copy.

Empty States

If user pauses, show "Need help answering?" guidance examples.

Success States

Interview summary card with editable assumptions and confirmed facts.

Error States

Generation failure fallback to static question set without data loss.

Responsive Behaviour

Sticky response box on mobile; transcript panel collapses into drawer.

Accessibility

Prompt text read order and explicit role labels for conversational elements.

User Flow

Prompt loop → Clarification → Summary confirmation → Twin synthesis.

Dashboard

Purpose

Provide strategic command center for decisions, actions, and twin health.

Wireframe Description

Top insights row, decision queue, simulation summary, and action column.

Components

Twin health card, next actions list, decision queue cards, insight highlights.

Buttons

Primary: Run simulation; Secondary: Update twin; Utility: View full report.

Inputs

Filters for timeframe, goals mode, and simulation status.

Animations

Card reorder animation on priority changes and subtle KPI count-up.

Transitions

Deep-link navigation into simulations, timeline, and insights modules.

Loading States

Skeleton rows for decision queue and chart placeholders.

Empty States

New-user dashboard shows onboarding completion CTA and first simulation prompt.

Success States

Action completion toast and progress streak indicator.

Error States

Partial module failure surfaces in-card retry without blocking entire dashboard.

Responsive Behaviour

Cards stack by priority with sticky top actions on mobile.

Accessibility

Readable summary labels and keyboard order matching visual priority.

User Flow

Open dashboard → identify priority → take action → review outcome.

Career Timeline

Purpose

Visualize trajectory history and future branch possibilities over time.

Wireframe Description

Horizontal timeline with expandable milestones and future branch lanes.

Components

Milestone nodes, branch overlays, confidence bands, notes drawer.

Buttons

Primary: Add milestone; Secondary: Compare branches; Utility: Toggle assumptions.

Inputs

Milestone date, role, impact score notes, branch filters.

Animations

Timeline scrub interpolation and branch highlight glow.

Transitions

Seamless handoff to simulation from selected branch node.

Loading States

Progressive rendering of timeline segments for faster first paint.

Empty States

No milestones state prompts "Add first turning point."

Success States

Milestone saved and integrated into twin recalibration queue.

Error States

Date conflicts and invalid sequence warnings with correction hints.

Responsive Behaviour

Timeline converts to vertical stacked list on narrow widths.

Accessibility

Alternative table view for non-pointer users and screen readers.

User Flow

Review past → inspect branch → run scenario from branch context.

Twin Profile

Purpose

Expose full digital twin composition and confidence quality.

Wireframe Description

Profile summary header, dimension tabs, source provenance panel.

Components

Skill matrix, motivation map, risk appetite gauge, evidence ledger.

Buttons

Primary: Recalibrate twin; Secondary: Edit assumptions; Utility: View version diff.

Inputs

Editable goals, constraints, strength adjustments, preference toggles.

Animations

Dimension chart transitions when switching profile tabs.

Transitions

Version picker transition to historical twin snapshots.

Loading States

Deferred loading of deep analytics modules with shimmer placeholders.

Empty States

If dimension has low data, prompt user for targeted enrichment step.

Success States

Assumption updates show confidence uplift summary.

Error States

Conflict alerts when edits contradict locked imported fields.

Responsive Behaviour

Tab group collapses into segmented control; ledger in slide-up panel.

Accessibility

Graph values mirrored in text tables; tab semantics applied.

User Flow

Inspect twin → adjust assumption → rerun simulation.

Simulation Engine

Purpose

Create and execute structured career scenario simulations.

Wireframe Description

Scenario builder on left, assumptions panel center, run output right.

Components

Scenario cards, assumptions editor, output charts, recommendation module.

Buttons

Primary: Run simulation; Secondary: Save scenario set; Utility: Duplicate scenario.

Inputs

Salary floor, geography, timeline, learning intensity, risk tolerance sliders.

Animations

Output chart morph animation between scenario runs.

Transitions

Builder mode to results mode with fixed anchor summary bar.

Loading States

Multi-step compute status: validating, simulating, calibrating, finalizing.

Empty States

No scenario created state with templates and "start from baseline" option.

Success States

Results generated banner and recommendation snapshot availability.

Error States

Assumption conflict errors and compute timeout fallback prompt.

Responsive Behaviour

Scenario builder becomes full-screen step flow on mobile.

Accessibility

All charts include text equivalents and keyboard-controllable filters.

User Flow

Define scenarios → run compute → inspect outputs → send to comparison.

Scenario Comparison

Purpose

Help users decide by exposing path tradeoffs side by side.

Wireframe Description

Column comparison cards with weighted metric sliders and summary verdict panel.

Components

Scenario columns, radar chart, regret view, weighted score bar.

Buttons

Primary: Choose path; Secondary: Adjust weights; Utility: Save comparison.

Inputs

Importance sliders for salary, growth, balance, leadership, risk.

Animations

Real-time score recalculation animation on slider changes.

Transitions

Transition to action plan generator from selected scenario.

Loading States

Short recalculation loader for weighted score update requests.

Empty States

Prompt to run at least two simulations before comparing.

Success States

Decision lock-in confirmation and next-step experiment creation.

Error States

If metrics unavailable, show partial comparison with visible caveat.

Responsive Behaviour

Horizontal swipe cards on mobile with pinned summary footer.

Accessibility

Slider values announce changes and affective labels.

User Flow

Select scenarios → tune priorities → choose path → create execution plan.

Career Insights

Purpose

Deliver ongoing strategic guidance from twin and market shifts.

Wireframe Description

Insight feed with weekly digest card, blind spots, and opportunity alerts.

Components

Insight cards, urgency tags, trend sparkline, recommendation actions.

Buttons

Primary: Apply insight; Secondary: Dismiss; Utility: Why this insight?

Inputs

Insight filters by type, urgency, and source confidence.

Animations

Card entry transitions and urgency pulse for critical items.

Transitions

Deep links to simulation or profile edit from each insight card.

Loading States

Staggered insight feed loading with placeholder cards.

Empty States

"No new insights yet" state with suggestion to run a fresh simulation.

Success States

Applied insight feedback updates dashboard action queue.

Error States

Insight service outage banner with last cached digest fallback.

Responsive Behaviour

Feed becomes single-column with sticky filter drawer on mobile.

Accessibility

Urgency tags include textual meaning beyond color cues.

User Flow

Review insight → validate rationale → apply to action plan.

Reports

Purpose

Package decisions and simulations for mentors, peers, or stakeholders.

Wireframe Description

Report template selector, content inclusion checklist, preview panel.

Components

Template cards, privacy controls, preview canvas, share settings.

Buttons

Primary: Generate report; Secondary: Share link; Utility: Download PDF.

Inputs

Template choice, audience type, redaction toggles, link expiry selector.

Animations

Preview refresh shimmer and generation progress ring.

Transitions

Draft-to-final mode transition with immutable snapshot timestamp.

Loading States

Report generation progress with stage labels.

Empty States

No saved simulation state prompting run requirement before report creation.

Success States

Report ready dialog with copy link and download actions.

Error States

Generation fail state with retry and simplified export fallback.

Responsive Behaviour

Preview collapses below controls; actions pinned at bottom.

Accessibility

Preview summaries are screen-reader consumable in structured text.

User Flow

Select template → configure privacy → preview → generate/share.

Settings

Purpose

Allow users to control data, privacy, preferences, and integrations.

Wireframe Description

Category tabs: Profile, Integrations, Privacy, Notifications, AI Preferences.

Components

Toggle groups, permission cards, account actions, export/delete controls.

Buttons

Primary: Save changes; Secondary: Revoke source; Danger: Delete account.

Inputs

Preference toggles, language/timezone, communication settings, consent checkboxes.

Animations

Toggle feedback micro animation and confirmation slide-in panel.

Transitions

Settings tab content crossfade preserving scroll position per tab.

Loading States

Optimistic toggle state with rollback on save failure.

Empty States

No integration connected state with clear connect actions.

Success States

Settings saved toast with undo for reversible changes.

Error States

Permission revoke or delete failures displayed with support guidance.

Responsive Behaviour

Tab list becomes accordion on mobile for easier scanning.

Accessibility

All toggles have explicit labels and state announcements.

User Flow

Open settings → adjust controls → save → verify effect in product.
