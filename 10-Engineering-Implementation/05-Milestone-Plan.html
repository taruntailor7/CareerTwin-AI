Critical Path

M3 → M4 → M5 → M6 → M7 defines the product-critical sequence. Delays here directly impact first usable simulation delivery.

  • Frontend and report tracks can progress in parallel after M6.
  • Reliability work should start earlier as a shadow track, not only at M10.

Cross-Milestone Risk Mitigations

  • Introduce quality gates at each milestone (API contract, test, security, AI eval).
  • Use feature flags for every high-risk subsystem release.
  • Run weekly architecture and prompt governance review.
  • Maintain visible burn-up with dependency risk tracking by engineering manager.
