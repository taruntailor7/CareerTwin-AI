Product Vision Understanding

CareerTwin AI aims to create a new category: career decision intelligence, not resume optimization or generic AI coaching.

  • Core promise: simulate likely career futures before irreversible decisions.
  • Core trust stance: evidence-backed recommendations with uncertainty, never deterministic prediction.
  • Core business thesis: longitudinal digital career twin becomes defensible product moat.

User Personas

Inflection Engineer

Choosing between IC depth, management, startup, or AI pivot path.

Global Mover

Balancing relocation opportunities with uncertainty and personal constraints.

Skilled Drifter

High capability but low strategic clarity and inconsistent long-term planning.

Aspiring Founder

Evaluating founder route versus career stability and income certainty.

Complete User Journey

  1. Discovery and trust framing (what this product is and is not).
  2. Twin Genesis onboarding (profile + imports + interview).
  3. Twin synthesis with confidence layers.
  4. Simulation setup and scenario comparison.
  5. Decision conversion into short experiments.
  6. Weekly insight loop and twin evolution updates.
  7. Report sharing with mentors/family/stakeholders.

Career Twin Lifecycle

  • Genesis: initial model generated from imported + interviewed signals.
  • Calibration: assumptions corrected, confidence re-weighted.
  • Decision Support: simulations tied to versioned twin snapshots.
  • Evolution: ongoing updates from user actions and new context.

AI Workflow Understanding

  • Ingestion and enrichment from multiple sources.
  • Context builder composes memory + assumptions + constraints.
  • Simulation and reasoning pipelines generate structured outputs.
  • Evidence and confidence engines gate recommendation quality.
  • Safety/policy layer filters final responses.

Simulation Workflow Understanding

  1. User defines 2-5 scenarios with constraints.
  2. Engine computes multi-metric outputs and confidence bands.
  3. Scenario comparison ranks tradeoffs based on user-weighted priorities.
  4. System converts recommendation into practical next-step experiment plan.

Dashboard & Reports Understanding

  • Dashboard is the decision command center (next actions, pending decisions, twin health).
  • Insights feed drives longitudinal engagement and timing decisions.
  • Reports transform simulations into shareable decision briefs.

AI Interview Understanding

  • Adaptive questioning targets ambiguity and contradiction gaps.
  • Responses classified by certainty class (confirmed/inferred/uncertain).
  • Interview output feeds directly into twin confidence recalibration.

Data Flow Understanding

Frontend → API gateway → domain services → DB/cache/storage + async workers → insight/report artifacts.

  • Heavy operations (parsing, simulation, reports) are asynchronous.
  • Versioned twin state is a key dependency for reproducible outputs.
  • Traceability is expected via audit events and evidence references.

Integration Understanding

  • Resume parsing pipeline
  • LinkedIn profile import
  • GitHub activity/metadata import
  • Portfolio crawling/parsing
  • Potential future channels: email, analytics, enterprise partner APIs

Technical Constraints

  • High trust bar: outputs must be explainable and uncertainty-aware.
  • Latency pressure on simulation UX despite complex compute pipeline.
  • Security/privacy requirements are central due to sensitive career data.
  • Scope pressure: rich product ambition vs hackathon/beta execution limits.
